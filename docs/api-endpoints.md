# مرجع مسارات API — صوري

> توثيق شامل لجميع نقاط واجهة برمجة التطبيقات في صوري.

---

## 1. شكل الاستجابة الموحد

### النجاح

```json
{
  "data": { ... },
  "message": "رسالة اختيارية بالعربية"
}
```

### الخطأ

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "رسالة الخطأ بالعربية"
  }
}
```

### رموز الأخطاء

| الرمز              | الحالة | الوصف                          |
| ------------------ | ------ | ------------------------------ |
| `VALIDATION_ERROR` | 400    | فشل التحقق من المدخلات         |
| `UNAUTHORIZED`     | 401    | توكن مفقود أو غير صالح         |
| `FORBIDDEN`        | 403    | غير مصرح بتنفيذ الإجراء        |
| `NOT_FOUND`        | 404    | المورد غير موجود               |
| `CONFLICT`         | 409    | تعارض (مثل بريد مُسجّل مسبقًا) |
| `SERVER_ERROR`     | 500    | خطأ غير متوقع في الخادم        |

---

## 2. المصادقة

**نموذج الجلسة:** بعد تسجيل الدخول أو التسجيل، يُعيّن الخادم **Cookie HttpOnly** باسم `auth-token` يحمل الـ JWT (لا يُعاد التوكن في جسم الاستجابة). المتصفح يُرسِل الـ cookie تلقائيًا مع طلبات نفس النطاق. مسارات API المحمية تقرأ التوكن من الـ **cookie أولاً**، ثم من رأس `Authorization: Bearer <token>` كبديل (اختبارات أو عملاء برمجيون)، ثم تتحقق من `sessionVersion` المخزّن للمستخدم لمنع استخدام الجلسات القديمة/المُبطلة.

**تسجيل الخروج:** `POST /api/auth/logout` يمسح الـ cookie من الخادم ويُبطل الجلسة الحالية عبر رفع `sessionVersion` (عند توفر جلسة صالحة).

---

### POST /api/auth/register

إنشاء حساب جديد. يُعيّن **Set-Cookie** (`auth-token`) ويُرجع بيانات المستخدم فقط (بدون توكن في الجسم).

| الحقل           | النوع  | مطلوب | الوصف             |
| --------------- | ------ | ----- | ----------------- |
| name            | string | نعم   | 3–50 حرف          |
| email           | string | نعم   | صيغة بريد صحيحة   |
| password        | string | نعم   | 6 أحرف على الأقل  |
| confirmPassword | string | نعم   | مطابق لـ password |

**مثال طلب:**

```json
{
  "name": "أحمد محمد",
  "email": "ahmed@example.com",
  "password": "secret123",
  "confirmPassword": "secret123"
}
```

**استجابة ناجحة (201):** مع ترويسة `Set-Cookie: auth-token=...; HttpOnly; SameSite=Lax; ...`

```json
{
  "data": {
    "user": {
      "_id": "...",
      "name": "أحمد محمد",
      "email": "ahmed@example.com",
      "avatarUrl": null,
      "createdAt": "2026-01-01T00:00:00.000Z",
      "updatedAt": "2026-01-01T00:00:00.000Z"
    }
  },
  "message": "تم إنشاء الحساب بنجاح."
}
```

**أخطاء محتملة:** 400 (تحقق)، 409 (البريد مُسجّل مسبقًا)

---

### POST /api/auth/login

تسجيل الدخول بالبريد وكلمة المرور.

| الحقل    | النوع  | مطلوب |
| -------- | ------ | ----- |
| email    | string | نعم   |
| password | string | نعم   |

**استجابة ناجحة (200):** نفس شكل `register` (`data.user` فقط) مع `Set-Cookie` للجلسة.

**أخطاء محتملة:** 400 (تحقق)، 401 (بيانات الاعتماد غير صحيحة)

---

### POST /api/auth/logout

إنهاء الجلسة: يمسح cookie `auth-token` من الخادم. لا يتطلب جسمًا.

**استجابة ناجحة (200):** `{ "message": "تم تسجيل الخروج بنجاح." }`

---

### GET /api/auth/me

جلب بيانات المستخدم الحالي. يتطلب **جلسة صالحة**: cookie `auth-token` أو رأس `Authorization: Bearer <token>`.

**استجابة ناجحة (200):**

```json
{
  "data": {
    "_id": "...",
    "name": "أحمد محمد",
    "email": "ahmed@example.com",
    "avatarUrl": "https://...",
    "createdAt": "...",
    "updatedAt": "..."
  }
}
```

**أخطاء محتملة:** 401 (توكن مفقود أو منتهي)

## 3. الملف الشخصي

### PUT /api/profile

تحديث الاسم و/أو البريد. يتطلب جلسة.

| الحقل | النوع  | مطلوب | الوصف           |
| ----- | ------ | ----- | --------------- |
| name  | string | لا    | 3–50 حرف        |
| email | string | لا    | صيغة بريد صحيحة |

**أخطاء محتملة:** 400، 401، 404، 409 (البريد مُسجّل)

---

### PUT /api/profile/password

تغيير كلمة المرور. يتطلب جلسة. عند النجاح تُبطَل كل الجلسات السابقة للمستخدم (رفع `sessionVersion`) ويُمسح الـ cookie الحالي؛ يجب تسجيل الدخول مرة أخرى.

| الحقل           | النوع  | مطلوب |
| --------------- | ------ | ----- |
| currentPassword | string | نعم   |
| newPassword     | string | نعم   |
| confirmPassword | string | نعم   |

**أخطاء محتملة:** 400، 401 (كلمة المرور الحالية خاطئة)

---

### PUT /api/profile/avatar

رفع صورة الملف الشخصي. `multipart/form-data` مع حقل `avatar`. يتطلب جلسة. التحقق من النوع عبر **Magic Bytes** (مثل رفع المنشورات).

**أخطاء محتملة:** 400، 401، 413 (حجم الملف يتجاوز 2 ميجابايت)

---

### DELETE /api/profile/avatar

حذف صورة الملف الشخصي والعودة للافتراضي. يتطلب جلسة.

---

### DELETE /api/profile

حذف الحساب نهائيًا. يتطلب جلسة وكلمة المرور للتأكيد.

**مثال طلب:**

```json
{
  "password": "كلمة المرور الحالية"
}
```

**أخطاء محتملة:** 400، 401 (كلمة المرور خاطئة)، 404

---

## 4. الصور

### GET /api/photos

قائمة الصور العامة (مع pagination). اختياري: جلسة (cookie أو Bearer) لعرض حالة الإعجاب للمستخدم المسجّل.

**معاملات الاستعلام:**

| المعامل | النوع  | الافتراضي | الوصف                    |
| ------- | ------ | --------- | ------------------------ |
| page    | number | 1         | رقم الصفحة               |
| limit   | number | 12        | عدد العناصر (حد أقصى 50) |

**استجابة ناجحة (200):**

```json
{
  "data": [
    {
      "_id": "...",
      "title": "عنوان الصورة",
      "description": "الوصف",
      "imageUrl": "https://...",
      "user": {
        "_id": "...",
        "name": "أحمد",
        "avatarUrl": null
      },
      "likesCount": 5,
      "isLiked": false,
      "createdAt": "...",
      "updatedAt": "..."
    }
  ],
  "pagination": {
    "page": 1,
    "totalPages": 3,
    "total": 25,
    "limit": 12
  }
}
```

---

### POST /api/photos

رفع صورة جديدة. يتطلب جلسة (cookie أو Bearer). `multipart/form-data`:

| الحقل       | النوع  | مطلوب | الوصف                           |
| ----------- | ------ | ----- | ------------------------------- |
| title       | string | نعم   | 1–200 حرف                       |
| description | string | لا    | حتى 2000 حرف                    |
| photo       | File   | نعم   | PNG أو JPEG، حد أقصى 5 ميجابايت |

**التحقق من الملف:** الخادم يفحص **محتوى الملف (Magic Bytes)** وليس `Content-Type` المرسل من المتصفح فقط، لرفض الملفات المزوّرة.

**أخطاء محتملة:** 400، 401، 413

---

### GET /api/photos/mine

صور المستخدم الحالي. يتطلب جلسة. يدعم `page` و `limit`.

---

### PUT /api/photos/[id]

تعديل عنوان/وصف صورة. المالك فقط. يتطلب جلسة.

| الحقل       | النوع  | مطلوب |
| ----------- | ------ | ----- |
| title       | string | لا    |
| description | string | لا    |

**أخطاء محتملة:** 400، 401، 403، 404

---

### DELETE /api/photos/[id]

حذف صورة. المالك فقط. يتطلب جلسة.

**أخطاء محتملة:** 401، 403، 404

---

### POST /api/photos/[id]/like

تبديل الإعجاب (like/unlike). يتطلب جلسة.

**استجابة ناجحة (200):**

```json
{
  "data": {
    "liked": true,
    "likesCount": 6
  }
}
```

**أخطاء محتملة:** 401، 404

---

## 5. الصحة

### GET /api/health

فحص حالة التطبيق وقاعدة البيانات والتخزين.

**استجابة ناجحة (200):**

```json
{
  "status": "healthy",
  "database": "connected",
  "repositories": {
    "user": true,
    "photo": true,
    "like": true
  },
  "storage": {
    "type": "local",
    "healthy": true
  },
  "timestamp": "2026-03-14T12:00:00.000Z"
}
```

**استجابة متدهورة (503):** عند فشل قاعدة البيانات أو التخزين.

## 6. ملخص سريع

عمود **Auth:** «جلسة» = cookie `auth-token` (المتصفح) أو `Authorization: Bearer` (بديل).

| Method | Path                  | Auth    | الوصف               |
| ------ | --------------------- | ------- | ------------------- |
| POST   | /api/auth/register    | —       | إنشاء حساب + cookie |
| POST   | /api/auth/login       | —       | تسجيل دخول + cookie |
| POST   | /api/auth/logout      | —       | مسح الجلسة (cookie) |
| GET    | /api/auth/me          | جلسة    | بيانات المستخدم     |
| PUT    | /api/profile          | جلسة    | تحديث البيانات      |
| PUT    | /api/profile/password | جلسة    | تغيير كلمة المرور   |
| PUT    | /api/profile/avatar   | جلسة    | رفع صورة شخصية      |
| DELETE | /api/profile/avatar   | جلسة    | حذف الصورة الشخصية  |
| DELETE | /api/profile          | جلسة    | حذف الحساب          |
| GET    | /api/photos           | اختياري | قائمة الصور العامة  |
| POST   | /api/photos           | جلسة    | رفع صورة            |
| GET    | /api/photos/mine      | جلسة    | صور المستخدم        |
| PUT    | /api/photos/[id]      | جلسة    | تعديل صورة          |
| DELETE | /api/photos/[id]      | جلسة    | حذف صورة            |
| POST   | /api/photos/[id]/like | جلسة    | تبديل الإعجاب       |
| GET    | /api/health           | —       | فحص الصحة           |
